Whats the difference between git and GitHub?

Git is used to edit and work on code/files locally via the terminal. GitHub is a powerful tool for team collaboration where branches can be made (or in the terminal) to edit code, submit these changes as pull requests for review, then merge back to the master branch where the project takes its final form.

What is Master?

Master branch is the where the reviewed, working, collaborated version of a project is kept. It isn’t edited directly.

Why create a Branch?

To safely make changes or additions to a projects code without the risk of bugging the master branch.
